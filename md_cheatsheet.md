# Markdown Cheatsheet

## **Fonts**
&nbsp; 

### **Headers**: # = h1, ## = h2, ...etc
&nbsp;  

### ***Italics***: & * OR _ (wrap around text)
&nbsp;
 

### **Bold**: ** OR __ (wrap around text) &nbsp;  
&nbsp;

### **~~Strikethrough~~**: ~~ (wrap around text)
&nbsp;

---
&nbsp;
## **Lists**
&nbsp;

### ULs: prefix with * OR -
* Example

&nbsp;

### **OLs**: prefix with 1 (will increment automatically)
1. Example
1. Example

&nbsp;

### **Nested Lists**: Use indent, alternate * and -
* Parent list
    - Child list
        * Grandchild list

&nbsp;


---
&nbsp;
## **Links and Images**
&nbsp;

### **URLs**: Use <URL\> if it's just a URL. If not, use [Title\](URL)
* Example 1: <https://cnn.com>
* Example 2: [CNN](https://cnn.com)

&nbsp;

### **URLs (variable format)**: [name][variable]. At the bottom of the MD page, list [variable] : URL (like a footnote.)
* Example: [NPR][1]

&nbsp;

### **Images**: ![Title\](URL)
* Example: ![Cat Time](https://www.seriousfacts.com/wp-content/uploads/2017/05/calico-cat.jpg)

&nbsp;

---
&nbsp;
## **Block Quotes and Code Blocks**
&nbsp;

### **Block Quotes**: >
> Example of a block quote 

&nbsp;

### **Code Blocks**: indent with a tab
    console.log('Example of a code block')

&nbsp;

### **Inline Code highlighting**: ` (wrap around content)
* Example: This is an `inline code` highlight

&nbsp;

---
&nbsp;
## **Github Only**
&nbsp;

### **Checkboxes**: *[ ] Text
* Can add an "X" in the [ ] if something is completed
